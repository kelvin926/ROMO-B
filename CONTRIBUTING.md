# Contributing

- Ubuntu 22.04 and ROS 2 Humble are the only supported host combination.
- Work directly on `main` as agreed for this private laboratory repository.
- Make small commits only after the relevant local checks pass; never force-push.
- Fix a failed remote check with a new commit instead of rewriting history.
- Do not commit secrets, device-specific IP addresses, bags, PCDs, maps, or build outputs.
- Update `docs/status.md` whenever a hardware assumption becomes verified or disproved.
- Preserve physical E-stop access and use the suspended-wheel procedure for first motion.
